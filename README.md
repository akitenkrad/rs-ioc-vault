# rs-ioc-vault

`rs-ioc-vault` は，OSINT 由来の脅威インジケータ (IoC) を正規化・重複排除して 1 つの SQLite ファイルに集約する，軽量な IoC ストアです．単一バイナリ `ioc-vault` (CLI) と公開ライブラリ `rs-ioc-vault` (ファサード) を提供し，URLhaus / ThreatFox / CISA KEV などのフィードを取り込み，複合条件での検索・時間減衰スコアリング・STIX 2.1 / MISP / CSV / JSONL でのエクスポートを行えます．STIX 2.1 Cyber-observable 命名に準拠しているため，TIP / SIEM / SOAR との相互運用が容易です．

## クレート構成

| クレート | 役割 |
|----------|------|
| `ioc-vault-core` | I/O を持たないドメインモデル (IoC 型, 正規化, dedup ハッシュ, 検索クエリ) |
| `ioc-vault-store` | SQLite (sqlx) 永続化・検索・減衰適用 |
| `ioc-vault-collect` | コレクタトレイトと収集コンテキスト |
| `ioc-vault-adapters` | 個別フィードのアダプタ (URLhaus / ThreatFox / CISA KEV) |
| `ioc-vault-export` | CSV / JSONL / STIX 2.1 / MISP へのエクスポート |
| `rs-ioc-vault` | 上記を束ねる公開ファサード (`IocVault`) |
| `ioc-vault-cli` | 単一バイナリ `ioc-vault` |

## CLI の使い方

```bash
# 初期化 (open + migrate)
ioc-vault init --db ~/.ioc-vault/vault.db

# ソース管理
ioc-vault source list
ioc-vault source add demo --url https://example/demo --feed-type csv --confidence 80
ioc-vault source enable urlhaus
ioc-vault source disable threatfox

# 全ソース更新 (直近 7 日分) / 特定ソース更新
ioc-vault update --all --since 7d
ioc-vault update --source threatfox --since 2026-04-01

# クイック照会
ioc-vault lookup 203.0.113.42
ioc-vault lookup evil.example.com --format json

# 複合検索
ioc-vault search \
    --type ipv4 \
    --source urlhaus,threatfox \
    --threat-type c2 \
    --since 30d \
    --min-confidence 70 \
    --cidr 203.0.113.0/24 \
    --format json

# 全文検索 (FTS5)
ioc-vault search --fts "emotet OR qakbot" --limit 100

# エクスポート (--out 省略時は標準出力)
ioc-vault export --format stix --since 7d --out feed.json
ioc-vault export --format misp --threat-type phishing --out phishing.misp.json
ioc-vault export --format csv --type ipv4 --out ipv4.csv
ioc-vault export --format jsonl --min-confidence 80

# 時間減衰スコアの再計算
ioc-vault decay

# 統計情報
ioc-vault stats
```

`export` は `search` とまったく同じフィルタフラグ (`--type` / `--source` / `--threat-type` / `--malware-family` / `--tag` / `--cve` / `--since` / `--min-confidence` / `--min-decay` / `--cidr` / `--regex` / `--contains` / `--prefix` / `--exact` / `--fts` / `--limit` / `--order`) を受け付けます．`--format` には `csv` / `jsonl` / `stix` (`stix2`, `stix2.1` も可) / `misp` を指定できます．

## ライブラリとしての利用

```rust
use rs_ioc_vault::{ExportFormat, IocVault, SearchQuery};

# async fn run() -> anyhow::Result<()> {
let vault = IocVault::builder()
    .database("vault.db")
    .with_default_collectors()
    .build()
    .await?;

let query = SearchQuery::builder().min_confidence(70).limit(100).build();
let stdout = std::io::stdout();
let n = vault.export(ExportFormat::Stix, &query, stdout.lock()).await?;
println!("exported {n} records");
# Ok(())
# }
```
